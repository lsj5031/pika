## 2026-01-13T00:40:27.137Z - US-003
Session Discovery Implementation (US-003):

**Session Discovery Pattern:**
- Scan .pi/agent/sessions/ subdirectory in each configured project root
- Parse session.jsonl files (JSONL format - one JSON object per line)
- Return Vec<SessionInfo> with all discovered sessions

**SessionInfo Struct:**
- id: String (session ID from sessionId field)
- project_path: PathBuf (path to the project containing the session)
- name: String (human-readable name, falls back to ID if empty)
- created_at: String (timestamp from JSON, defaults to "Unknown" if empty)
- is_active: bool (currently hardcoded to false, will be enhanced later)

**Graceful Degradation:**
- Missing .pi directories: Return empty Vec instead of error
- Missing session.jsonl: Return empty Vec instead of error
- Malformed JSONL lines: Skip and continue parsing (don't fail entire scan)
- Empty lines in JSONL: Skip gracefully

**JSONL Parsing Pattern:**
- Use BufReader with BufRead for line-by-line reading
- serde_json::from_str for each line
- SessionEntry struct with #[serde(rename)] for field name mapping
- Trim and skip empty lines before parsing

**Error Handling:**
- SessionError enum with IoError and ParseError variants
- Use thiserror for clean error derive macros
- Errors propagated via Result<> for file operations
- Public API (scan_sessions) never fails - returns empty Vec on errors

**Module Organization:**
- sessions.rs for session discovery logic
- SessionEntry private struct for internal parsing
- SessionInfo public struct for API responses

**WebSocket Implementation (US-004):**

**Axum WebSocket Integration:**
- Axum's WebSocket support uses tokio-tungstenite under the hood
- WebSocketUpgrade extractor upgrades HTTP to WebSocket connection
- Message::Text expects Utf8Bytes, not String - use .into() for automatic conversion
- split() separates WebSocket into Sender and Receiver for concurrent operations

**Broadcast Channel Pattern:**
- tokio::sync::broadcast::channel for pub/sub to multiple clients
- tx.subscribe() creates a new receiver that gets all future messages
- Channel capacity (100) provides backpressure if clients are slow
- Use _ to ignore send errors when no clients are connected

**Connection Management:**
- UUID for unique client identification
- Arc<RwLock<HashSet<String>>> for thread-safe active client tracking
- Clone values before moving into async closures if needed later
- tokio::select! waits for first completed task (cleanup on disconnect)

**State Management:**
- WSState must be cloned and passed via Router::with_state
- Clone is cheap due to Arc wrapping of internal state
- State accessible in handlers via State<T> extractor

**Gotchas:**
- Ping/Pong handling with split socket is complex - simplified for now
- Unused functions generate warnings but are expected (API for future stories)
- Edition "2024" works fine with axum 0.8 WebSocket features
