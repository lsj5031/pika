Session Discovery Implementation (US-003):

**Session Discovery Pattern:**
- Scan .pi/agent/sessions/ subdirectory in each configured project root
- Parse session.jsonl files (JSONL format - one JSON object per line)
- Return Vec<SessionInfo> with all discovered sessions

**SessionInfo Struct:**
- id: String (session ID from sessionId field)
- project_path: PathBuf (path to the project containing the session)
- name: String (human-readable name, falls back to ID if empty)
- created_at: String (timestamp from JSON, defaults to "Unknown" if empty)
- is_active: bool (currently hardcoded to false, will be enhanced later)

**Graceful Degradation:**
- Missing .pi directories: Return empty Vec instead of error
- Missing session.jsonl: Return empty Vec instead of error
- Malformed JSONL lines: Skip and continue parsing (don't fail entire scan)
- Empty lines in JSONL: Skip gracefully

**JSONL Parsing Pattern:**
- Use BufReader with BufRead for line-by-line reading
- serde_json::from_str for each line
- SessionEntry struct with #[serde(rename)] for field name mapping
- Trim and skip empty lines before parsing

**Error Handling:**
- SessionError enum with IoError and ParseError variants
- Use thiserror for clean error derive macros
- Errors propagated via Result<> for file operations
- Public API (scan_sessions) never fails - returns empty Vec on errors

**Module Organization:**
- sessions.rs for session discovery logic
- SessionEntry private struct for internal parsing
- SessionInfo public struct for API responses

HTTP API Endpoints Implementation (US-006):

**Axum Router State Management Pattern:**
- When merging routers with .merge(), they must have the same state type
- Router<ApiState> cannot be merged with Router<AppState> - state types must match
- Solution: Keep ApiState as public struct in api.rs, have AppState wrap it
- Access nested state via State<AppState> then dereference (state.api_state.config)

**API Module Architecture:**
- ApiState wraps shared resources with Arc<RwLock<ProjectConfig>> for thread-safe mutable access
- Response structs (ProjectResponse, SessionResponse) need Serialize/Deserialize derives
- ErrorResponse implements IntoResponse for custom HTTP status codes (NOT_FOUND, BAD_REQUEST, etc.)
- Use Result<Json<T>, ErrorResponse> for handlers that can fail

**State Nesting Pattern:**
- AppState combines multiple module-specific states (ws_state, api_state)
- Individual modules (api, websocket) have their own state types
- AppState wraps ApiState rather than duplicating config access logic
- Handlers extract State<AppState> and access nested state (state.api_state.config.read().await)

**REST API Design Patterns:**
- GET /api/projects - list all projects with session counts
- GET /api/projects/:id/sessions - nested routes for hierarchical resources
- GET /api/sessions - list all sessions across all projects  
- GET /api/sessions/:id - individual resource access by ID
- Hash-based project IDs from paths for stable identification

**Move vs Clone Gotchas:**
- Strings don't implement Copy trait
- When moving String into struct field but also need it later as lookup key, use .clone()
- Pattern: let id = generate_id(); Struct { id: id.clone(), ... use id for lookup ... }

**Acceptance Criteria Met:**

- Use `<ScrollYView>` wrapper for vertical scrolling

- ScrollYView uses `height: Fill` to take available space
